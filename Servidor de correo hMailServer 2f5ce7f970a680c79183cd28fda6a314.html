Servidor de correo hMailServer

Configuración de recursos necesarios para la práctica

Configuración de red y Firewall del servidor

Configuramos la tarjeta de red en Adaptador puente con la configuración IP dada

Con el comando ipconfig /all veremos la configuración de la red del servidor como se muestra en la captura.

Desactivamos el firewall del servidor para realizar esta práctica, no recomendable en el entorno real

Iremos a Panel de control>Sistema y seguridad>Activar o desactivar el firewall y lo desactivamos.

Instalamos y administramos el servidor DNS

Instalamos el rol servidor DNS desde el Administrador del servidor de Windows Server

Iremos a Administrar>Agregar roles y características>Roles de servidor y activamos servidor DNS.

Instalamos la característica .NET Framework 3.5 para compilar lenguajes de programación que usaremos

Esto lo haremos desde Características tras los Roles de servidor, activamos la opción que se ve en la captura.

Especificamos una ruta de acceso de origen alternativa para agregar los roles que faltan que vienen en la iso de instalación de nuestra máquina Windows Server

Por tanto introduciremos la iso de Windows Server 2019, le daremos a Especifique una ruta de acceso de origen alternativa y dentro establecemos la ruta D:\sources\sxs en la cual tenemos los roles por añadir, luego le damos a Instalar.

Comienza la instalación de el rol servidor DNS y la instalación de las características de .NET Framework 3.5

Esperaremos a que termine la instalación, en caso de dar error comprobar que la iso introducida es la correcta para los roles alternativos de los que hemos hablado antes.

Creación de la zona directa sri16.edu en la que estableceremos los registros en el siguiente paso

Crearemos la zona como lo hacíamos en las prácticas de DNS en Windows pero crearemos una zona directa simple con el nombre sriXX.edu que en mi caso es sri16.edu

Ahora creamos unos registros establecidos para que el cliente pueda apuntar a ellos correctamente y por tanto responda al servidor DNS

Creamos los registros pop, pop3, email, correo, smtp, imap y www que apunten hacia la IP de nuestro servidor (192.168.1.235 en mi caso), los cuales tendremos que comprobar, tanto en servidor como en cliente.
